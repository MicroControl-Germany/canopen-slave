cos404ao.h File Reference

Detailed Description

This module implements the variables and callback functions for the object dictionary from CiA 404 (analog outputs).

General implementation of this module is as follows: 1) Via CANopen SDO objects (x300h) for process values (PV) user set target analogue output value. 2) This value will be scaled to the field value (FV). Scaling parameter are provided by user implemented functions McDacGetGain() and McDacGetOffset(). 3) With this parameter PV will be scaled to FV using formula y=m*x+b, the resulting FV is set in SDO objects (x330h) and passed to application by calling McDacSetFV16().

void Cos404_AO_Error ( uint8_t  ubEnableV)

Set / Reset Error Condition.

Parameters
ubEnableV- set / reset error condition

This function sets / resets the digital output module error condition. Passing a 0 to ubEnableV resets the error condition, passing a 1 to ubEnableV sets the error condition. If the error condition is set, the output is set according to the object 6340h.

◆ Cos404_AO_Process()

void Cos404_AO_Process ( uint8_t  ubChannelV)

Process selected AO channel.

Parameters
ubChannelV- Analog output channel

This function is typically called cyclic by an event to process analog outputs depending on set values.

Finally this value triggers application dependent McDacSetFV16() function to provide AO value for corresponding channel.

Attention
This function should be called after the COS 404 module have been initialised by Cos404_AO_ParmInit()

Parameters
ubSubIndexVsub-index
ubReqCodeVread / write access
