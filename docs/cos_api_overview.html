CANopen (FD) Slave API overview

All header files and source files of the CANopen (FD) Slave protocol stack package are located inside the source directory. The following picture shows an overview of the source code structure.

Directory structure

Source code overview

The files inside the source/application directory shall be used to adopt the CANopen (FD) Slave protocol stack to the target. All other files shall not be modified by the user.

Component Directory

Description

CANopen (FD) Slave Code Examples application/examples-cos

Examples for evaluation boards
 

CANopen (FD) Slave Templates application/template-cos

Examples for callback functions, object dictionary and stack initialisation
 

CANpie FD API canpie-fd
device

Responsible for reading and writing CAN (FD) messages between the CANopen (FD) Slave protocol stack and the CAN (FD) bus

Hardware Abstraction Layer hal
misc

Compiler data type definition and optional hardware abstraction layer
 

CANopen (FD) Slave Protocol Stack stack-cos

Implementation of the CANopen (FD) Slave protocol stack
 

Optional add-on modules for the CANopen (FD) Slave protocol stack are located inside the source/stack-cos directory.
