cos_mgr.h File Reference

Detailed Description

This module holds all functions for the initialisation and control of the CANopen slave. Some functions of this module need to be adopted to the target platform:

  • CosMgrGetBitrate()
  • CosMgrGetNodeId()
  • CosMgrGetSerialNumber()
  • CosMgrOnBusOff()

These functions are located in the cos_user.c file.

Initialise the CANopen Slave.

Parameters
ubCanIfV- Physical CAN interface
uwConfigV- Initial configuration
Returns
Error Code

The function returns eCOS_ERR_NONE on success. It is responsible for the following tasks:

  • Initialise the CANopen state machine
  • Initialise LED management

Possible values for uwConfigV are:

  • COS_CONF_SLAVE for Classical CANopen
  • COS_CONF_FD for CANopen FD

◆ CosMgrStart()

uint8_t CosMgrStart ( uint8_t  ubNodeIdV,
int32_t  slBitrateNomSelV,
int32_t  slBitrateDatSelV 
)
Parameters
ubNodeIdV- Device node-ID (NID)
slBitrateNomSelV- Nominal bit rate value
slBitrateDatSelV- Data bit rate value
Returns
Error Code

This routine starts the CANopen protocol stack on the given node-ID and bit rate. The initialisation routines from other CANopen services are called and the Boot-up message is sent. The function returns eCOS_ERR_NONE on success.

In case the device shall be started in classical CANopen mode the parameter slBitrateDatSelV is set to eCP_BITRATE_NONE.

Attention
The number and position of parameters has been changed with major version 6 (CANopen FD) of the protocol stack.
